Unpacking Operator¶

  • The unpacking operator can be used to get the actual values from any iterable object.
  • To iterate over a string or a list or a range, you use a single asterisk, *, followed by the object name.
  • To iterate over a dictionary, you use two asterisks, **, followed by the dictonary name.
    • When unpacking multiple dictionaries together, if there is more than one dictioary that has the same key, the value used in the last dictionary in the unpacking will provide the value for that key.
In [ ]:
# List Unpacking
numbers = [1,2,3,4,5]
# Using the * will put all remaining items into a new list named, other
first, second, *other, last = numbers

print("First:",first)
print("Second:",second)
print("Other:",other)
print("Last:",last)

# Range Unpacking
values = [*range(5)]
print("Unpacked Range:",values)

# Unpacking multiple objects together
new_object = [*range(5),*"Hello"]
print("Combined Objects Unpacked together:",new_object)

# Combine lists with unpacking
list1 = [1,2,3]
list2 = ['a',4,'c']
list3 = [*list1, *list2]
print("Combined Lists with Unpacking:",list3)

# You can also put other objects between your unpacking lists
list4 = [*list1, "a", "e", *list2, *"hello"]
print("Multiple Objects Unpacked:",list4)

# Dictionary Unpacking
dict1 = {"x":1, "y":2}
dict2 = {"x":10, "m": 0}
print("Unpacked Dictionary:",{**dict1})
# OR set the unpacked dictionary to a new dictionary and add values with it.
added_to_dict = {**dict1, "z":3}
print("Added valus to unpacked dictionary:",added_to_dict)

# Combine two dictionaries with unpacking. x will be 10 since even though it is in both dictionarys, it is 10 in the last dictionary added in the unpacking statement. 
print("Unpacked Dictionaries with identical keys:",{**dict1, **dict2})

Map Function¶

****Most python users prefer List Comprehension to Map function. They get same result.**

  • It takes 2 arguments
    • The function you want to apply to each item
    • The list of items -It returns an iterable that can be used with a for loop or list function to get the results for each item.

items = [
  ("Product1", 10),
  ("Product2", 8),
  ("Product3", 12)
]

# Using a for loop to iterate over the resutling mapped iterable

for item in prices:
  print(item)

# Using the list function to make a list from the results of the map iterable.
prices2 = list(map(lambda item: item[1], items))
print(prices2)

Filter Function¶

Most python users prefer List Comprehension to Filter function. They get same result.

  • Filter function applies the given function to each item of the iterable and if the item matches some criteria, it will return it.
  • It takes 2 arguments
    • The function you want to apply to each item
      • The results of this function should be a boolean that returns true or false when applied to the items in the list/iterable
    • The list of items
  • It returns an iterable that can be used with a for loop or list function to get the results for each item.
In [ ]:
items = [
  ("Product1", 10),
  ("Product2", 8),
  ("Product3", 12)
]

# Using a for loop to iterate over the resutling filtered iterable. Returns the filtered results one by one
filtered = filter(lambda item: item[1] >= 10, items)
for item in filtered:
  print(item)

# Using the list function to make a list from the results of the filtered iterable. Returns a list with the filtered results.
filtered2 = list(filter(lambda item: item[1] >= 10, items))
print(filtered2)

Set Operations¶

  • Union of two sets

    • To make a union of two sets, we use the | symbol between two sets.
    • This will return a new set that includes all the items that are in EITHER the first set or in the second set

  • Intersection of two sets

    • To get the intersection of two sets, we use the & symbol between two sets.
    • This will return a new set that includes all the items that are in BOTH the first set AND the second set

  • Difference between two sets

    • To get the difference between two sets, we use the - symbol between two sets.
    • This will return a new set that includes all the items in the first set that aren't in the second set.

  • Semantic Difference between two sets

    • To get the semantic difference between two sets, we use the ^ symbol between two sets.
    • This will return a new set that includes the items that are either in the first or second set, but not both.
In [ ]:
numbers = [1,1,2,3,4]
first = set(numbers)
second = {1,5}

# Union of two sets
print("Union of Sets:",(first | second))

# Intersection of two sets
print("Intersection of Sets:",(first & second))

# Difference between two sets.
print('What is in the first set only:',(first - second))
print('What is in the second set only:',(second - first))

# Sematnic difference between two sets. 1 is the only number that is in both sets, so will be every number in the two sets except for 1.
print("Semantic Difference:",(first ^ second))

Dictionaries¶

  • To get the index of a dictionary value, you must use the key, using the format:
    • dictionary["key"] <-You must put the quotes around the key.
  • Add a key using the format:
    • dictionary['key'] = value
  • Delete a dictionary key/value pair using the format: -del dictionary['key']

Looping over Dictionaries¶

      • for key_value in dictionary.items(): print(key_value) <- Returns a tuple of each key/value pair

Data Classes and Named Tuples¶

  • Data classes are classes that do not contain any methods, they are just used data.
  • When comparing if two instances of a data class are equal, you get False no matter even if the attributes provided are the same because it is comparing on their location in memory, not the values. To prevent this, it's better to use Named Tuples then data classes.
  • To use Named Tuples, you must include the following important statement at the top of your code
    • from collections import namedtuple
  • When creating the named tuple type, it has two required arguments:
    1. The name of this new type we want to create
    2. An array of attribute names (the attributes we want these instances to have)
  • The format of the named tuple is:
    • VariableName = namedtuple("VariableName",[attribute1, attribute2, etc.]) <-the word namedtuple is the actual word you must enter there, not an example of a value. The 'VariableName' and 'VariableName' should be the same.
  • The VariableName uses pascal naming convention because it represents a type, like a class.
  • When passing the arguments to create the instance of these namedtuples, you must use keyword arguments, meaning you must use the format: name of the attribute equals the value, not just entering the value.
    • Example:
      • It SHOULD look like p1 = Point(x=1, y=2)
      • It CANNOT just be p1 = Point(1,2)
  • Named tuples are immutable, so once you create an object from them, you cannot change the values of the attributes and would need to make a new instance to get a different value.
  • When comparing named tuples instances, we do not have to create magic methods to show what is equal. If the two named tuple object have the exact same attributes, they will return True for equal with no other work needed.

Comparison Magic Methods¶

  • If you tried to compare if two instances of a class were equal without setting a comparison magic method, you would always return False because it would be comparing their locations in storage which wouldn't be the same. That's what comparison magic methods allow you to define.
  • The comparison magic methods are:
    • __eq__ stands for equal to and provides the definition of what would make two instances of this class equal when compared
    • __gt__ stands for greater then and provides the definition of what would make one instance of this class greater then another instance of this class.
    • __lt__ stands for less then and provides the definition of what would make one instance of this class less then another instance of this class.
  • The format for these methods when defined in the class are (replace eq with gt or lt and < or > if using them, same concept though):
    • Equal sign shown in format is two equal signs together
    def __eq__(self, other)
  return self.x == other.x and self.y == other.y
  • If you define the __gt__ magic method for a class, it will automatically also know how to define a less then comparison, __lt__ without having to define it in the class. Works with having either __gt__ or __lt__ defined, don't have to have both.
  • If you set the __str__ magic method for a class and compare the string results of two instances of that class, you could get True without having to define the comparison magic methods, but otherwise you will need to use them.

class Point:
  def __init__(self, x, y):
    self.x = x
    self.y = y
    
  def __str__(self):
    return f"({self.x},{self.y})"
  
  def __eq__(self, other):
    return self.x == other.x and self.y == other.y
  
  def __gt__(self, other):
    # Can use just greater then, or can use greater than and equal to, or a combination of both as shown below. 
    return self.x > other.x and self.y >= other.y

Making Custom Container/Data Type Classes¶

  • You can create custom classes that have a built-in data structure, like lists, dictionaries, etc, but with improved abilities.
  • You can use the add method to adapt what is needed or done when adding items to your data structure.
    • Example: Can create an add method that allows you to add to a dictionary but is not case sensitive like a normal dictionary. (See Example in code below)
  • To be able to get the values of your data structure in the way you want, you can define a __getitem__ magic method.
    • This magic method takes two parameters when creating it in the class: self, key
  • To be able to set the values of your data structure in the way you want, you can define a __setitem__ magic method.
    • This magic method takes three parameters when creating it in the class: self, key, value
  • To be able to get the number of items in your class's data structure in the way you want, you can define a __len__ magic method.
    • This magic method takes 1 parameter: self
  • To be able to iterate over hte items in your data structure, you can define a __iter__ magic method.
    • This magic method takes three parameters when creating it in the class: self

class TagCloud:
  def __init__(self):
    self.tags = {}
    
  # This allows you to add items to a dictionary, but they are not case-sensitive if you use different capital letters when doing the keys and will not return an error if you add a key that doesn't exist. It will just create a new key with 1 value.
  def add(self, tag):
    self.tags[tag.title()] = self.tags.get(tag.title(), 0) + 1

  # This shows that you can set the way that you get the value of the keys in your dictionary.
  def __getitem__(self, tag):
    # The get() method for a dictionary has the first parameter which is the key, and the second parameter which is the value, and if there is no value, it sets that as new key to a value of 0 by default.
    return self.tags.get(tag.title(), 0,)
  
  # This shows that you can set the value of a key in your dictionary. Again, with no problem of case-sensitivity.
  def __setitem__(self, tag, count):
    self.tags[tag.title()] = count
  
  # This allows you to get the number of items in your class created dictionary when using the len method for an instance of your class.
  def __len__(self):
    return len(self.tags)

  # If we didn't have this magic method, you would get an error when attempting to iterate over your class created dictionary items. Turns your data structure into an iterable object using the iter() function.
  def __iter__(self):
    return iter(self.tags)

      • When using this method of property creation, you define the attribute in the class normally, so in the format:
        • self.attributeName = attributeName
      • For the GET function of the property: You add a @property decorator above a getter function to make it the GET function for an attribute. You need to name the function the same name as the attribute.
        • Format:
          @property
def attributeName(self):
  (more code to adjust if needed...)
  return self.attributeName
      • For the SET function of the property: You add a @attributeName.setter decorator above the setter function to make it the SET function for an attribute. You need to name the function the same as the attribute. It will know the difference from the GET function by the decorator.
        • Format:
          @attributeName.setter
def price(self, value):
  (Code to adjust control attribute setting if needed...)
  self.__attributeName = value
      • You don't need both a getter and a setter function when creating a property. If you just define a getter for your attribute, you will have a read-only attribute that can't be edited after the initial setting when making the instance of the class.

    2. The second method of property creation: You can define a property function that has 4 OPTIONAL parameters: The format of the function is
      • attributeName = property(fget, fset, fdel, doc)
        • fget <- call to the function used to get the value of the attribute
        • fset <- call to the function used to set the value of the attribute
        • fdel <- call to the function used to deleting the attribute (not just the value of it)
        • doc <- documentation for this property.
      • We can then use the name of the property with dot notation to get, set, or delete the attribute from your instance of the class.

        • Example: product.price = 10 will set the product attribute to a value of 10.
      • When using this method, you need to change the attribute definition in the class to look like this format: -self.set_attributeName(attributeName) instead of self.attributeName = attributeName

Inheritence¶

  • Inheritence is a mechanism that allows us to define the common behaviors or common functions (attributes can be inherited too) in one class and then inherit them in other classes.
  • Example:
    • All animals have to eat. So if you have an animal class, whih would be called the Parent (or Base) class, you can then also have a mammal class and a fish class (called Child or Sub classes). Both these classes can inherit the eat method since they are still animals.
  • Inheritence has 3 huge benefits:
    • We don't have to duplicate code in multiple classes.
    • If there is a bug, we only have to change the code in one place to resolve it.
    • If we need to change what the method does, we only have to change it one place.
  • The format for Inheritence with classes is:
    class Parent:
  def methodOne(self):
    code for method...

  def methodTwo(self):
    code for method...

class Child(Parent):
  def methodThree(self):
    code for method...
  • When a child inherits a parent class, it will get access to all the methods and attributes of that class.
  • You do not write the Parent class's inherited methods and attributes in the child class. They are just automatically available when creating instances of that child class.

  • If multiple of the inherited bases classes have the same method in them, the first base class that is used as a parameter in the child class that is inheriting them is the one whos method is called.

Method Overriding¶

  • Method overriding means replacing or extending a method defined in the base class.
  • If you have an __init__ method (constructor) in the base class and you add a constructor to the child class, it will remove any of the attributes that were accessible in the base class because it overrides them.
  • If we still want to have the attributes from the base class, in the constructor of the child class, use the built in super() function to get access to the parent (or base) class.
  • You can use super() to extend any method in a class, not just the __init__ constructor method.
  • You can add the super() method to the beginning or end of the child class's method to adjust when it applies the parent class's method to that of the child.

# Parent (or Base) Class
class Game():
  def __init__(self):
    self.players = 4
  
  def roll(self):
    print('Roll Dice')
    
  def timer(self):
    print('Set Timer for 30 seconds')
    
  def play(self):
    print("Play")

# Child (or Sub) Class
class Card(Game):
  def shuffle(self):
    print('Shuffle')
    
# Child (or Sub) Class
class Board(Game):
  def __init__(self):
    # Since the Board class has an __init__ method, it would override the __init__ in the base class, Game, which means Board would not be able to access the players attribute either. The super() method allows us to access the __init__ method from the base class without overriding it. 
    super().__init__()
    self.boardType = 'path'
  
  def roll(self):
    super().roll()
    print('Roll 2nd Dice')
    
  def timer(self):
    print('Set Timer for 1 min')
    super().timer()

  def move(self):
    print('Move')
    
candyland = Board()
# Even though we overrode the __init__ method with our constructor in the Board class, since we used super() we can still access the player attribute.
print("Number of Players from the Game Parent class:",candyland.players)

# We extended the abilities of the roll method of the super class by adding a roll method to our child class and putting the super method for the base class with it. 
candyland.roll()

# You can use the super() at the begging or end of the child class's method. It will change when it is completed.
# This method set the timer in the child class first and then the timer in the base class since the super() method was done at the end of the child's identical timer method. 
candyland.timer()

Abstract Base Classes¶

  • An abstract base class is a base classes that is used to provide some code that is going to be reused (inherited) across multiple child classes.
  • You CANNOT create an instance of an abstract base class.
  • Abstract Base Classes allow us to also create a common interface across multiple sub classes.
  • To use an abstract base class, you must import from abc import ABC, abstractmethod.
  • To make a class an abstract base class, you need to have the class inherit the ABC class you imported
  • To make the common interface of the abstract base class, you need to put a decorator in front of any method that will be required in any child classes that inherit the abstract base class. The method just has to have the same name. It is not required to have the same actual code. The decorator you must use is @abstractmethod
  • If you try to make a clas that inherits an abstract base class and do not implement all the required methods in it (the ones with the @abstractmethod on them), the new child class will also be abstract and you will not be able to create an instance of it.

Polymorphism¶

  • Polymorphism means many forms. This concept allows you to create a method that can have many different forms(Aka types) and those forms are determined at run time.
  • When using polymorphic behavior, you don't need to have a base class to make it work or for the child classes to inherit from that class. As long as all the classes have the function you are going to use on them, that is all you need.

from abc import ABC, abstractmethod

class UIControl(ABC):
  
  @abstractmethod
  def draw(self):
    pass
  
class TextBox(UIControl):
  def draw(self):
    print("TextBox")

class DropDownList(UIControl):
  def draw(self):
    print("DropDownList")

def draw(control):
  control.draw()
    
# This function has a for loop in it so it can iterate over multiple controls (which are instances of the classes above) and execute the draw method on each individually and show all results. 
def drawMultiple(controls):
  for control in controls:
    control.draw()

tb = TextBox()

draw(ddl)

Packages¶

  • A package is a container for one or more modules.

  • In file system terms, a package is mapped to a directory and a module is mapped to a file

  • If you create a new sub-directory (folder) that contains modules, you need to create a file in the package named __init__.py which will tell Python to treat this sub-directory folder as a package. The file will be empty.

  • To access the functions in one of the modules in this package from another module not in the sub-directory package, you need prefix the name of the module you are importing with the name of the package.

    • Example: from packageName.moduleName import functionName1, functionName2
      • Then in the code, you could call functionName1 without any other names having to be added.

  • Another way to do the same thing is you can also import the module as an object, from the package and then use notation to get the functions you want from the module in the code.

    • Example: from packageName import moduleName
      • Then you would use the moduleName with dot notation to call the funtions from the module.
        • Example: moduleName.functionName1

  • To create sub-packages is the same process. You would create a folder in a folder and then to turn it into a sub-package, you must add the __init__.py file and the desired modules.

  • When you then want to import from the sub-package you must use the format: from packageName.subPackageName import module.

    • You can do the same thing to get the latest minor version of version 2: pip install packageName==2.*
    • If the minor version is the same, you must first uninstall the entire package manually before then trying to reinstall the specific minor version package with the bug fixes you want.
    • An alternate way to install the latest version of a specific minor package is to use the format: pip install requests~=2.9.0 <- This uses the tilda, ~, and will find the latest compatible version to the specific one we included, such as 2.9.2

Openpyxl¶

  • This package lets you work with excel spreadsheets using python. Install it in your enviornment with pipenv install openpyxl
  • To use this package, you must use the following import statement in your code:
    • import openpyxl
  • To work with a spreadsheet, we can do 1 of 2 things:
    1. Create an empty workbook in memory
    2. Load an existing workbook
  1. workbookObject = openpyxl.Workbook() - Use this to create an empty workbook object with one sheet.
  2. existingWorkbookObject = openpyxl.load_workbook("/path/to/spreadsheet/file.xlsx") - Use this to add an existing workbook to a workbook object.
    • existingWorkbookObject.sheetNames - This attribute of your workbook object returns a LIST of the names of all the sheets in the workbook.
    • workbookObject["Sheetname"] - Use this to access the specific sheet in your workbook. The sheetName is case sensitive, so must be spelled exactly as it is in the workbook.
    • workbookObject.create_sheet("NewSheetName", indexNumber) - Use this to create a new sheet at the end of our workbook. The two arguments are:
      1. The name of the new sheet
      2. The index where you want to add the sheet. (Starts at 0)
    • workbookObject.remove_sheet(sheetObject) - Use this to remove a sheet object from your workbook.

  • There are two ways to work with specific cells, the two methods are:

    1. Square bracket Notion

      • cellObject = sheet["a1"] - This will reference cell A1 of the sheet object you used. It creates a cell object.
      • cellObject.value - Attribute that gets the contents of the specific cell
      • cellObject.row - Attribute that returns what row the cell is in.
      • cellObject.column - Attribute that returns what column the cell is in.
      • cellObject.coordinate - Attribute that return the coordinate of the cell.
        • Example: A1

    2. .cell(row=Number, column=Number) - Use this as another way to access the cell. Make sure to use keywork arguments so you know which is row and which is column.

    • You can also do .cell(row, 1) - This will access all the cells in row 1. Same can be done with column.

    • Choice 1 is faster, but choice 2 allows you to iterate over all the rows and all the columns if you want to dynamically access the cells.

    • sheetObject.max_column - Attribute to get the number of columns in the sheet

    • sheetObject.max_rows - Attribute to get the number of rows in the sheet

    • columCells = sheetObject["a"] - Gets all the cells in the specific column. This returns a tuple of all the cell objects.

    • newCellsObject = sheetObject["columnLeter:columnLetter"] - This gets all the values of the cells between row A and C. This will return a tuple of tuples. Each inner tuple will represent a single column.

    • sheetObject["coordinate1:coordinate2"] - This will get a refernce to all the cells between the cell at coordiante1 and the cell at coordinate2.

    • sheetObject[rowNumber] - This will get all the cell references for row 1

    • sheetObject[rowNumber1:rowNumber2] - This will get the cell references for all cells between rowNumber1 and rowNumber2

    • sheetObject.append() - This allows you to add a new row to the end of the sheet. The parameters need to be either a tuple or list that contains the contents of the new row seperated by commas.

    • sheetObject.insert_rows( indexNumber) - To insert a row at a given index.

    • sheetObject.insert_cols(indexNumber") - To insert a blank column at a given index.

    • sheetObject.delete_rows(indexNumber) - To delete a row of your sheet

    • sheetObject.delete_cols("indexNumber) - To delete a column of your sheet

  • To save the workbook as a new file, use:

    • workbookObject.save("/path/to/newSpreadsheetFile.xlsx")
